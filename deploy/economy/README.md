# NOVA 最低成本算力方案

核对日期：2026-07-17。

## 当前结论

首选“本机 Lite 控制面 + 魔搭免费 A10 Notebook”：

1. 本机运行 Vue、FastAPI、SQLite 与 Mock，GPU 成本为 0。
2. 魔搭 Notebook 提供 A10 24GB 和 `/mnt/workspace` 下 100GB 持久化空间，先完成私有图片 + WAV 到 MP4 的真实纵切。
3. EchoMimicV3 Flash 官方显存要求为 12GB，A10 24GB 可承载首轮 512×512、81 帧、8 步推理。
4. 实时数字人继续使用 Mock；真实低延迟链路会要求 GPU 常驻，验证需求前不启用。

运行包和具体步骤见 [deploy/modelscope/README.md](../modelscope/README.md)。

## 为什么不直接建公开创空间

Notebook 更适合授权人物和声音的私有质量验证。xGPU 创空间需要申请权限，并且应用代码与发布边界需要单独审计。纵切通过后，再把推理入口包装成 Gradio xGPU Worker；xGPU 按请求动态调度并在计算结束后释放 GPU，可继续保持低闲置成本。

## 备选顺序

1. 魔搭 Notebook 免费 A10：开发和质量基线首选。
2. 魔搭 xGPU 创空间：获得权限后用于按请求演示。
3. Modal Serverless L4：魔搭额度不足、且需要稳定 API 时使用。
4. RunPod A5000/3090 Pod：只有稳定连续批处理能充分占用实例时使用。

## 成本与安全护栏

- 免费额度、排队、实例时长以魔搭账户页面实时信息为准，不在代码中硬编码承诺。
- 首轮输出固定 512×512、25fps、最多 81 帧、单并发。
- 使用者必须确认人物与声音授权；不要把私密素材发布到公开 Studio。
- 模型缓存留在持久盘，完成推理后停止 GPU 实例。
- 正式上线前固定源码 commit、模型文件哈希、许可证与依赖清单。
- 远端状态未知时禁止自动重复提交，避免重复任务和不可控资源占用。
